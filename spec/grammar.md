# Smalltalk Grammar Specification
**Version 3.1.0**

The canonical type reference for the `.st` format. Every entry type, every field, every symbol.

---

## Syntax

Every `.st` entry follows one rule:

```
TYPE: identifier | field | field | field
```

- One entry per line
- `TYPE` is always uppercase
- `identifier` is the unique key for the entry
- Lines beginning with `#` are comments
- Append new entries — never overwrite existing ones

---

## Separators

| Symbol | Meaning | Example |
|---|---|---|
| `\|` | Field boundary | `RULE: brand \| no-purple \| hard` |
| `+` | Multiple values in one field | `when:next+vite+remix` |
| `:` | Key-value pair within a field | `broke:token-refresh` |
| `>` | Choice over alternative | `railway>vercel` |

Values: lowercase-hyphenated, no spaces inside field values.

---

## Universal Types

Used in any `.st` file — memory, skills, agents, or palace.

```
RULE:     id | rule-description | hard|soft
REF:      id | path/to/file.st  | covers:topic
NOTE:     id | observation
CONFIG:   id | key | value
CONTEXT:  id | scope | value
DECISION: id | choice>rejected | reason | date
PATTERN:  id | broke:what | cause:why | fix:what | reuse:y/n
```

### Field notes

**RULE** — `hard` = enforced always. `soft` = default, can be overridden with reason.

**DECISION** — `choice>rejected` encodes the decision direction. The thing after `>` is what was rejected. Usually append date as `YYYY-MM`.

**PATTERN** — `broke:` is what failed. `cause:` is root cause. `fix:` is the resolution. `reuse:y` means this pattern fires proactively on similar tasks.

**REF** — links to the full `.md` document for depth-2 loading. Agent reads `.st` first; reads `.md` only when the specific topic is needed.

## Memory Types

For tracking what worked, who prefers what, what components exist, and what prompts produced exceptional output.

```
WIN:       id | technique | outcome | repeat:y/n
CLIENT:    id | pref:what | avoid:what | updated:date
COMPONENT: id | stack | does:what | use-when
PROMPT:    id | task-type | approach | why-worked | reuse:y/n
```

---

## Knowledge Graph Types

Temporal entity-relationship graph. Human-readable, git-trackable, no database.

```
LINK:   source | rel:relationship | target | [valid_from:YYYY-MM] | [ended:YYYY-MM] | [stability:x]
TUNNEL: wing-id | connects:wing-id | via:shared-topic
```

### Common `rel:` values

```
works-on       deployed-to     member-of
depends        blocks          assigned-to
defined-as     contributes-to  reports-to
references     replaced-by     caused-by
```

### Temporal fields

Temporal fields work on **any** entry type, not just `LINK`.

| Field | Meaning |
|---|---|
| `valid_from:YYYY-MM` | When this fact became true |
| `ended:YYYY-MM` | When this fact stopped being true |
| `stability:permanent` | Always load — core truth, never auto-expire |
| `stability:stable` | Valid until explicitly ended (default) |
| `stability:transient` | Expected to change — assignments, sprints, current status |

> **Hard rule:** `ended:` is never set by a clock or auto-expiry. It is set only by the contradiction resolution cycle: `smalltalk check` → `smalltalk kg invalidate` → `smalltalk check` confirms clear.

## Skill Types

For encoding agent skills, methodologies, and templates — replaces SKILL.md files.

```
SKILL:   id | triggers | stack | version
USE:     id | when:context
PHASE:   id | number | name | what-happens
STEP:    id | phase | number | action | required|optional
STACK:   id | layer | tool | why
CHECK:   id | verification-item | required|optional
AVOID:   id | antipatterns+separated
SCRIPT:  id | path/to/script | what-it-does
```

### Field notes

**SKILL** — `triggers` is a `+`-separated list of keywords that cause this skill to be routed. Matched by `smalltalk route`.

**USE** — declares the triggering context. `smalltalk_route` uses `USE when:` fields as content-scoring signals.

**STEP** — `phase` links to a `PHASE` entry. Steps within a phase are ordered by `number`. `required` vs `optional` controls strictness.

## Agent Types

For encoding autonomous agent specifications, tasks, and orchestration logic.

```
AGENT:   id | role | capabilities | scope
TASK:    id | action | target | priority:high|mid|low
TRIGGER: id | event | condition | then:action
OUTPUT:  id | type | destination | format
ERROR:   id | broke:what | cause:why | state:recovered|unresolved
```

## Palace Types

For the `_index.st` file generated by `smalltalk palace init`. Do not write these by hand — they are auto-generated.

```
WING:   id | type:project|person|topic | keywords:kw1+kw2+kw3
ROOM:   id | wing:wing-id | hall:TYPE1+TYPE2 | file:relative/path.st
TUNNEL: id | wings:wing-a+wing-b | topic
LAYER:  id | file:path|trigger:event | load:always|source:x|command:x
```

### Hall values

A room's `hall:` field tags it with the entry types it contains:
- `facts` — RULE, CONFIG, CONTEXT
- `events` — DECISION, TRIGGER
- `discoveries` — PATTERN, WIN, ERROR
- `preferences` — CLIENT, NOTE
- `advice` — SKILL, USE, PHASE, STEP

## What To Compress

| Content type | Action |
|---|---|
| Skill definitions, `.cursorrules`, routing files | Compress → `.st` |
| Memory logs (decisions, patterns, wins) | Compress → `.st` |
| Agent specs, pipeline definitions | Compress → `.st` |
| Design systems, style guides | Compress → `.st` |
| Config and environment mapping | Compress → `.st` |
| Human-maintained context (`stack.md`, `CONTEXT.md`) | Keep as `.md` |
| Reference docs loaded on demand | Keep as `.md`, link via `REF` |
| READMEs and public documentation | Keep as `.md` |
| Code files, MCP servers, API schemas | Never compress |

**Rule:** If an agent loads it every session → compress. If only when a specific topic arises → keep `.md` + `REF`. If it must be syntactically exact → never compress.
