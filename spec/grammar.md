# Smalltalk Grammar Spec
> Version 3.0.1

Smalltalk is a compressed token-efficient dialect for AI agent communication.
Every LLM that reads text understands it — no decoder, no fine-tuning, no cloud API required.
One line per entry. Pipe-delimited. Typed prefix. Lowercase-hyphenated values.

---

## Syntax

```
TYPE: identifier | field | field | field
```

| Element | Rule |
|---|---|
| `TYPE` | Uppercase prefix — defines the entry category |
| `identifier` | The skill, agent, or subject this entry belongs to |
| `fields` | Pipe-delimited values, as short as unambiguous |
| Values | Lowercase hyphenated — no spaces inside field values |
| `+` | Multiple values within one field: `next+vite+remix` |
| `:` | Key-value within a field: `broke:vite-build` |
| `>` | Choice over alternative: `cloudflare>vercel` |

One entry per line. No line breaks inside an entry.
Append entries — never overwrite.
`.st` files contain Smalltalk only — no markdown headers, no prose.

---

## Severity Flags

```
hard      → non-negotiable, always apply
soft      → preferred, apply when possible
required  → must complete
optional  → complete if relevant
high      → urgent priority
mid       → normal priority
low       → parked / background
```

---

## Universal Types
Apply to any file — memory, skills, agents, config.

```
RULE:     identifier | rule-description | hard|soft
REF:      identifier | path/to/file.st  | covers:topic
NOTE:     identifier | observation-or-caveat
CONFIG:   identifier | key | value
CONTEXT:  identifier | scope | value
DECISION: identifier | choice>rejected | reason | date
PATTERN:  identifier | broke:what | cause:why | fix:what | reuse:y/n
```

> **Wake-up filter:** `smalltalk wake-up` loads DECISION (all active), RULE (hard only — soft excluded), PATTERN (all active), WIN (repeat:y only). All other types are available on demand via `navigate()` and `search()`.

## Memory Types
For `_brain/` subfolders and long-term knowledge logs.

```

CLIENT:    identifier | pref:what | avoid:what | updated:date
COMPONENT: identifier | stack | does:what | use-when
PROMPT:    identifier | task-type | approach | why-worked | reuse:y/n
```

## Palace Types
For navigation — structure .st files into wings, rooms, and tunnels.
Generated by `smalltalk palace init`. Read by `smalltalk_navigate()`.

```
WING:   identifier | type:project|person|topic | keywords:kw1+kw2+kw3
ROOM:   identifier | wing:wing-id | hall:TYPE1+TYPE2 | file:relative/path.st
TUNNEL: identifier | wings:wing-a+wing-b | topic
LAYER:  identifier | file:path|trigger:event | load:always|source:x|command:x
```

WING declares a namespace (a project, person, or topic domain).
ROOM maps a topic to a file, within a wing, indexed by which TYPEs live there.
TUNNEL cross-references the same room across two wings (auto-detected by palace init).
LAYER defines the memory stack — what loads always, on demand, or via search.

**Hall mapping** — MemPalace hall names translate to Smalltalk TYPEs:

| Hall | Smalltalk TYPEs |
|---|---|
| `hall_facts` | DECISION, RULE, CONFIG |
| `hall_events` | PATTERN, WIN |
| `hall_discoveries` | WIN, NOTE |
| `hall_preferences` | RULE (soft), CLIENT |
| `hall_advice` | PATTERN (reuse:y) |

Example `_index.st`:

```
LAYER:  L0 | file:identity.st | load:always
LAYER:  L1 | file:_index.st   | load:always
LAYER:  L2 | trigger:topic    | source:navigate
LAYER:  L3 | trigger:search   | command:smalltalk_search

WING:   myapp | type:project | keywords:myapp+analytics
WING:   kai   | type:person  | keywords:kai+backend

ROOM:   auth    | wing:myapp | hall:DECISION+PATTERN | file:projects/myapp/auth.st
ROOM:   auth    | wing:kai   | hall:WIN+PATTERN      | file:people/kai.st

TUNNEL: auth  | wings:myapp+kai | topic:auth
```

## Knowledge Graph Types
For entity relationships, temporal validity, and the contradiction→resolution cycle.

```
LINK:   source-entity | rel:relationship-type | target-entity | [valid_from:YYYY-MM] | [ended:YYYY-MM] | [stability:x]
```

LINK is the primary KG entry. The target is the first field without a known key prefix.
Good relation names: rel:works-on, rel:depends, rel:defines, rel:blocks, rel:reports-to,
                     rel:replaced-by, rel:caused-by, rel:assigned-to, rel:deployed-to

TUNNEL is a Palace type (see above) — auto-generated by `smalltalk palace init` when the same topic appears in multiple wings. It is not a KG entry.

## Temporal & Stability Fields
Optional fields that can appear on ANY entry type.

```
valid_from:YYYY-MM     When this fact became true
ended:YYYY-MM          When this fact stopped being true (set by contradiction resolution)
stability:permanent    Core truth — always load, only close via contradiction
stability:stable       Valid until explicitly ended (DEFAULT for all entries)
stability:transient    Time-windowed — expected to change frequently
```

**Critical rule:** `ended:` is NEVER set by a clock or scheduler.
It is set when a contradiction is resolved. This is the only correct update mechanism.

Resolution workflow:
  1. New fact arrives that contradicts existing active entry
  2. `smalltalk check` flags the contradiction (CRITICAL if stability:permanent)
  3. Add `ended:YYYY-MM` to the OLDER entry to close it
  4. New entry is now the single source of truth
  5. `smalltalk check` clears — no more contradiction

Historical entries (ended: <= today) are EXCLUDED from:
  - `smalltalk check` (contradiction scan)
  - `smalltalk wake-up` (system prompt context)
  - `smalltalk_navigate` results

Historical entries ARE included in:
  - `smalltalk_kg_timeline` (chronological story)
  - `smalltalk_kg_query(as_of=...)` (point-in-time query)
  - `smalltalk_search` (full history search)

## File Extension

Smalltalk files use `.st` extension.
Any `.md` agent file can be compressed to `.st`.
Original `.md` files can be kept as human reference — link them via `REF`.

**Two-layer classification:**
- **Brain / instructions layer** — compressible: `_brain/`, skill files, decisions, patterns, wins
- **Operations layer** — never compress: code files, MCP servers, API schemas, config with exact syntax

Rule: if a file must be syntactically exact to function → never compress it.

```

agent-name        for agent definitions
client-name       for client memory records
```

## Token Comparison

| Format | Lines | Approx tokens |
|---|---|---|
| Full markdown skill file | ~200 | ~1,800 |
| Skill + all sub-references | ~500+ | ~4,500+ |
| Smalltalk compressed | ~20 | ~180 |
| **Reduction** | **~90%** | **~90%** |

## Token Impact

| Approach | Context per session (50-room palace) | Monthly cost est. |
|---|---|---|
| Uncompressed, load all | ~20,000 tokens | ~$180/mo |
| Smalltalk compression only | ~2,000 tokens | ~$18/mo |
| Smalltalk + palace navigate | ~350 tokens | ~$3/mo |

The 350 tokens = `_index.st` (~150) + one relevant room file (~200).
Navigation means you only load the file that answers THIS question.
